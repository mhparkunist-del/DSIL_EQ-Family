# LIVE 모드 설정 가이드 (Google Sheets 연동)

DEMO 모드는 설정 없이 바로 동작합니다. 아래는 **여러 사람이 실제로 예약을 공유**하려는 경우의 설정입니다.

## 1. Google Sheet 만들기

1. [sheets.google.com](https://sheets.google.com) 에서 새 스프레드시트를 만듭니다. (이름: 예 `DSIL 장비예약`)

## 2. Apps Script 코드 붙여넣기

1. 시트 상단 메뉴 **확장 프로그램 → Apps Script** 클릭.
2. 기본 `Code.gs` 내용을 모두 지우고, 이 저장소의 **`apps-script/Code.gs`** 내용을 전부 붙여넣고 저장(💾).

## 3. 스크립트 속성 설정

1. Apps Script 편집기 좌측 **⚙ 프로젝트 설정** → 아래 **스크립트 속성**에서 두 개 추가:
   - `SHARED_SECRET` = 아무 긴 임의 문자열 (예: `dsil-7f3a9c2b...`) — 외부 노출 주의
   - `SALT` = 아무 임의 문자열 (예: `kaist-salt-2026`)

## 4. 시트 탭/헤더 자동 생성

1. Apps Script 편집기 상단 함수 선택 드롭다운에서 **`setupSheets`** 선택 → **실행(▶)**.
2. 최초 1회 권한 승인 팝업이 뜨면 본인 계정으로 허용합니다.
3. 시트로 돌아가면 `Equipment / Users / Reservations / UsageLogs` 4개 탭과 헤더,
   기본 관리자 계정(이름 `관리자`, 비번 `0000`)이 생성됩니다.

> 시트 컬럼(헤더) 스키마는 코드 하단 `setupSheets` 에 정의되어 있습니다. 컬럼 순서는 바꿔도 되지만
> **헤더 이름은 그대로** 두어야 합니다.

## 5. 웹앱으로 배포

1. 편집기 우측 상단 **배포 → 새 배포** 클릭.
2. 유형 선택(⚙) → **웹 앱**.
3. 설정:
   - **실행 주체(Execute as): 나(Me)**
   - **액세스 권한(Who has access): 모든 사용자(Anyone)**  ← "구글 계정 있는 모든 사용자" 아님!
4. **배포** → 표시되는 **웹 앱 URL**(`.../exec` 로 끝남)을 복사합니다.

> 이후 코드를 수정하면 **배포 → 배포 관리 → ✎ 편집 → 버전: 새 버전 → 배포** 로 갱신하세요.
> 그래야 URL이 그대로 유지됩니다. (새 배포를 만들면 URL이 바뀜)

## 6. 프론트엔드 연결

`js/config.js` 를 열어 두 값을 채웁니다:

```js
window.DSIL_CONFIG = {
  API_URL: "여기에 5단계에서 복사한 /exec URL 붙여넣기",
  SHARED_SECRET: "3단계의 SHARED_SECRET 과 동일한 값",
  ADMIN_PASSWORD: "0000",
  ORG_NAME: "KAIST DSIL",
};
```

저장 후 커밋·푸시하면 화면 우측 상단 뱃지가 **DEMO → LIVE** 로 바뀌고, 실제 시트에 데이터가 쌓입니다.

## 7. 동작 확인

- 예약 신청 → `Reservations` 탭에 행 추가, `Equipment` 상태가 `in_use` 로 변경되는지 확인.
- 같은 장비 재예약 시도 → **차단**(이전 사용자 실적 입력 필요).
- 실적 입력 → `UsageLogs` 탭에 행 추가, 장비가 다시 `available` 로.
- 두 브라우저 탭에서 동시에 같은 장비 예약 → **하나만 성공**(LockService 동시성 처리).

## 보안 참고 (솔직한 한계)

- 관리자 비번 `0000` 과 `SHARED_SECRET` 은 정적 프론트엔드에 포함되므로 이론상 추출 가능합니다.
  **내부 랩 도구 수준**의 보호이며 강한 보안이 아닙니다.
- 더 강한 보안이 필요하면 Google 로그인(OAuth) 기반 인증으로 전환해야 합니다.
- 사용자 비밀번호는 시트에 평문이 아닌 **SHA-256 해시**로 저장됩니다.
